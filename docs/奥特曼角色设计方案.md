# 奥特曼角色精灵图设计方案

## 技术要求

### 图片规格
- **文件格式**: PNG（支持透明背景）
- **图集尺寸**: 建议 256x256 像素（可扩展）
- **单个角色帧**: 32x32 像素
- **需要动画帧**: 每个角色需要8个方向的动画（上下左右+行走动画）

### 当前使用的精灵图
- 文件路径: `public/assets/32x32folk.png`
- 当前包含: 8个不同的卡通角色（f1-f8）

## 五位奥特曼角色设计要求

### 1. 迪迦奥特曼 (f1 - 第1行)
**外观特征**:
- 主体颜色: 红、紫、银三色
- 头部: V型额头彩色计时器（蓝色）
- 胸部: 圆形彩色计时器
- 体型: 匀称，复合型态
- 配色方案:
  - 上半身: 红色+紫色条纹
  - 下半身: 紫色+银色
  - 眼睛: 椭圆形黄色

**动画要求**:
- 上、下、左、右站立姿势
- 上、下、左、右行走动画（3帧循环）

### 2. 赛罗奥特曼 (f2 - 第2行)
**外观特征**:
- 主体颜色: 红、蓝、银三色
- 头部: 两个头镖（帕拉吉之盾的一部分）
- 胸部: 圆形彩色计时器
- 体型: 健壮，运动型
- 配色方案:
  - 头部: 银色带蓝色装饰
  - 上半身: 红色主体+蓝色条纹
  - 下半身: 蓝色+银色
  - 眼睛: 菱形黄色

**动画要求**:
- 上、下、左、右站立姿势
- 上、下、左、右行走动画（3帧循环）
- 可选: 头镖轻微闪光效果

### 3. 梦比优斯奥特曼 (f3 - 第3行)
**外观特征**:
- 主体颜色: 红、金、银三色
- 头部: 梦比姆气息（左手腕上的手环，可简化）
- 胸部: 圆形彩色计时器
- 体型: 年轻，灵活型
- 配色方案:
  - 上半身: 红色主体+金色条纹
  - 下半身: 红色+银色
  - 眼睛: 椭圆形黄色
  - 特征: 比其他奥特曼稍显年轻

**动画要求**:
- 上、下、左、右站立姿势
- 上、下、左、右行走动画（3帧循环）
- 动作可以更有活力

### 4. 艾斯奥特曼 (f4 - 第4行)
**外观特征**:
- 主体颜色: 红、银、白三色
- 头部: 星形标志（额头）
- 胸部: 圆形彩色计时器
- 体型: 稳重，力量型
- 配色方案:
  - 头部: 银色带星形装饰
  - 上半身: 红色主体+银色护甲
  - 下半身: 红色+白色
  - 眼睛: 椭圆形黄色

**动画要求**:
- 上、下、左、右站立姿势
- 上、下、左、右行走动画（3帧循环）
- 姿态沉稳

### 5. 泰罗奥特曼 (f5 - 第5行)
**外观特征**:
- 主体颜色: 红、蓝、金三色
- 头部: 奥特角（双角）- 最显著特征
- 胸部: 圆形彩色计时器
- 体型: 强壮，英雄型
- 配色方案:
  - 头部: 红色+金色双角
  - 上半身: 红色主体+蓝色、金色条纹
  - 下半身: 红色+蓝色
  - 眼睛: 椭圆形黄色

**动画要求**:
- 上、下、左、右站立姿势
- 上、下、左、右行走动画（3帧循环）
- 可选: 奥特角微微发光

## 像素艺术风格指南

### 风格要求
- **像素艺术风格**: 清晰的像素点，不要抗锯齿
- **色彩**: 鲜艳明亮，适合小朋友
- **轮廓**: 明显的黑色或深色轮廓
- **简化**: 在32x32的限制下，简化细节但保留特征

## 动画帧说明

每个方向需要3帧动画来实现行走效果：

1. **站立帧**: 基础姿势
2. **行走帧1**: 一只腿抬起
3. **行走帧2**: 另一只腿抬起

循环: 站立 → 行走1 → 站立 → 行走2 → 循环

## 实现步骤

### 方案A: 专业设计（推荐）
1. 聘请像素艺术家根据以上规格绘制
2. 提供奥特曼官方参考图
3. 确保每个角色特征明显可辨认
4. 测试32x32尺寸下的识别度

### 方案B: AI生成 + 手工优化
1. 使用AI生成工具（如Midjourney、DALL-E）生成奥特曼像素图
2. 提示词示例: "32x32 pixel art of Ultraman Tiga, red purple and silver colors, front view, pixel game sprite"
3. 手工调整和优化生成的图片
4. 确保透明背景和正确尺寸

### 方案C: 使用现有资源
1. 搜索开源奥特曼像素艺术
2. 购买或获得授权使用的精灵图
3. 根据游戏需求调整

## 替换步骤

1. 创建新的精灵图文件: `public/assets/32x32ultraman.png`
2. 更新 `data/characters.ts`:
```typescript
textureUrl: '/ai-town/assets/32x32ultraman.png',
```
3. 如果精灵布局改变,更新 `data/spritesheets/f1-f5.ts` 中的坐标
4. 测试所有角色动画是否正常
