# 本地后端使用指南

## 🎯 概述

本地后端系统让你可以在不依赖Convex云服务的情况下运行AI Town，完全免费，适合学习和测试。

## ✨ 特性

### 已实现功能

✅ **本地存储**
- 使用JSON文件存储记忆和角色数据
- 数据保存在 `./data` 目录
- 支持导出/导入数据

✅ **记忆系统**
- 三种记忆类型：对话、反思、关系
- 基于关键词的记忆检索（替代向量搜索）
- 多维评分：相关性 + 重要性 + 时效性
- 自动触发反思机制

✅ **LLM支持**
- 支持OpenAI API（可选）
- 内置模拟LLM（无需API Key）
- 自动回退机制

✅ **配置切换**
- 支持本地和Convex模式切换
- 通过环境变量配置
- 一键切换，无需修改代码

## ⚙️ 配置说明

### 环境变量

```bash
# 后端模式
BACKEND_TYPE=local          # 'local' 或 'convex'

# 本地存储配置
LOCAL_DATA_DIR=./data       # 数据目录路径

# LLM配置（可选）
OPENAI_API_KEY=sk-...       # OpenAI API Key
LLM_MODEL=gpt-3.5-turbo     # 模型选择

# Convex配置（仅convex模式需要）
VITE_CONVEX_URL=https://...
```

### 切换到Convex模式

编辑 `.env.local`:
```bash
BACKEND_TYPE=convex
VITE_CONVEX_URL=https://your-project.convex.cloud
```

重启应用即可。

## 🔧 API使用

### 基本用法

```typescript
import { localStorage } from './localBackend/storage';
import * as llm from './localBackend/llm';

// 添加角色
const player = localStorage.addPlayer({
  id: 'p1',
  name: '迪迦奥特曼',
  description: '温柔坚定的光之勇士',
  character: 'f1',
});

// 添加记忆
const memory = localStorage.addMemory({
  playerId: 'p1',
  playerName: '迪迦奥特曼',
  description: '与赛罗讨论了守护地球的使命',
  importance: 8,
  type: 'conversation',
  metadata: {
    otherPlayerId: 'p2',
    otherPlayerName: '赛罗奥特曼',
  },
});

// 搜索记忆
const memories = localStorage.searchMemories(
  'p1',                           // 角色ID
  ['赛罗', '地球', '守护'],      // 关键词
  5                               // 返回数量
);

// 生成对话
const response = await llm.generateConversationReply(
  '迪迦奥特曼',
  '温柔而坚定的战士',
  '赛罗奥特曼',
  ['你好', '最近怎么样'],
  memories.map(m => m.description)
);

// 获取统计
const stats = localStorage.getStats();
console.log(stats);
```

### 记忆检索算法

本地模式使用**关键词匹配 + 多维评分**：

```
综合评分 = 关键词匹配分 + 重要性分数 + 时效性分数

- 关键词匹配: 每个匹配关键词 +10分
- 重要性: 0-9分（创建时评估）
- 时效性: 0.99^小时数（指数衰减）
```

与Convex的向量搜索相比：
- ✅ 简单直观，易于理解
- ✅ 无需API调用，完全离线
- ⚠️ 语义理解能力较弱
- ⚠️ 只能精确匹配关键词

## 📊 数据格式

### 记忆数据 (Memory)

```typescript
{
  id: string;                    // 唯一ID
  playerId: string;              // 角色ID
  playerName: string;            // 角色名称
  description: string;           // 记忆描述
  importance: number;            // 重要性 (0-9)
  type: 'conversation' | 'reflection' | 'relationship';
  createdAt: number;             // 创建时间戳
  lastAccess: number;            // 最后访问时间
  metadata?: {                   // 元数据
    conversationId?: string;
    otherPlayerId?: string;
    otherPlayerName?: string;
    playerIds?: string[];
    relatedMemoryIds?: string[];
  };
}
```

### 角色数据 (Player)

```typescript
{
  id: string;                    // 角色ID
  name: string;                  // 角色名称
  description: string;           // 角色描述
  character: string;             // 角色类型
}
```

```typescript
localStorage.importData(convexData);
```


## 🐛 故障排除

### 问题1: 测试脚本报错

**错误**: `Cannot find module './storage'`

**解决**:
```bash
# 确保在项目根目录
cd /path/to/ai-town

# 使用正确的路径
npx tsx src/localBackend/test.ts
```

### 问题2: 数据文件未生成

**原因**: 权限问题或路径错误

**解决**:
```bash
# 手动创建数据目录
mkdir -p data

# 检查权限
ls -ld data/
```

### 问题3: LLM响应为空

**原因**: Mock模式的模式匹配失败

**解决**:
1. 配置真实的OpenAI API Key
2. 或修改 `mockLLM` 函数增加更多响应模式

### 问题4: 记忆检索结果不准确

**原因**: 关键词提取不够准确

**解决**:
- 手动指定关键词
- 优化 `extractKeywords` 函数
- 或升级到向量搜索方案
