## 系统架构概览

### 整体设计
AI Town的记忆系统采用了**向量数据库 + LLM生成**的混合架构，灵感来自斯坦福大学的"Generative Agents"论文。系统通过模拟人类记忆的存储、检索和反思过程，使AI角色具备长期记忆能力。

```
┌─────────────────────────────────────────────────────────┐
│                    记忆系统架构                          │
├─────────────────────────────────────────────────────────┤
│                                                          │
│  事件发生 → 记忆生成 → 向量化 → 存储                     │
│     ↓                                                    │
│  检索需求 → 向量搜索 → 多维排序 → 使用记忆               │
│     ↓                                                    │
│  累积阈值 → 触发反思 → 生成见解 → 高层记忆               │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

### 技术栈
- **数据库**: Convex (支持向量索引)
- **向量模型**: OpenAI text-embedding-ada-002 (1536维)
- **语言模型**: OpenAI GPT (用于对话总结和反思)
- **检索算法**: 向量相似度 + 多维评分排序

## 核心数据结构

### 1. memories 表
存储所有记忆的主表。

```typescript
{
  playerId: string,              // 记忆所属的角色ID
  description: string,           // 记忆的文本描述
  embeddingId: Id,               // 指向向量表的ID
  importance: number,            // 重要性评分 (0-9)
  lastAccess: number,            // 最后访问时间戳
  data: {                        // 记忆类型特定数据
    type: 'conversation' | 'relationship' | 'reflection',
    // ... 类型特定字段
  }
}
```

**索引设计**:
- `embeddingId`: 快速通过向量ID查找记忆
- `playerId_type`: 按角色和类型查询
- `playerId`: 按角色查询所有记忆

### 2. memoryEmbeddings 表
存储记忆的向量表示，支持语义搜索。

```typescript
{
  playerId: string,              // 角色ID (用于过滤)
  embedding: number[1536],       // 1536维向量
}
```

**向量索引**:
- `embedding`: 支持向量相似度搜索
- `filterFields`: [playerId] - 限制搜索范围到特定角色

### 3. embeddingsCache 表
缓存文本的向量表示，避免重复调用Embedding API。

```typescript
{
  textHash: bytes,               // 文本的哈希值
  embedding: number[1536],       // 对应的向量
}
```

## 记忆类型

系统支持三种类型的记忆，使用Union Type实现：

### 1. conversation (对话记忆)
记录角色之间的对话内容。

```typescript
{
  type: 'conversation',
  conversationId: string,        // 对话ID
  playerIds: string[],           // 其他参与者ID列表
}
```

**生成流程**:
1. 对话结束后触发
2. 收集对话中的所有消息
3. LLM从第一人称总结对话内容
4. 评估记忆重要性
5. 生成向量并存储

**示例**:
```
"与赛罗奥特曼在2024-02-02 20:00的对话：我们讨论了守护地球的使命，
赛罗分享了他在宇宙巡逻的经验。我很欣赏他的热血和勇气。"
```

### 2. reflection (反思记忆)
从多条记忆中提炼出的高层次见解。

```typescript
{
  type: 'reflection',
  relatedMemoryIds: Id[],        // 相关记忆的ID列表
}
```

**生成触发条件**:
- 最近100条记忆的重要性总分 > 500

**生成流程**:
1. 检查是否达到反思阈值
2. 收集最近100条记忆
3. LLM分析并生成3条高层见解
4. 为每条见解生成向量和重要性
5. 存储为反思记忆

**示例**:
```
"通过与多位战友的交流，我意识到团队协作比单打独斗更重要。
这种认知来自与赛罗、梦比优斯的多次对话。"
```

### 3. relationship (关系记忆)
记录对其他角色的看法和关系状态。

```typescript
{
  type: 'relationship',
  playerId: string,              // 关系对象的角色ID
}
```

**用途**: 存储角色间的动态关系（当前代码中未完全实现）

## 向量检索与排序

### 多维评分算法 (rankAndTouchMemories)

```typescript
// 文件: convex/agent/memory.ts:185-226
```

系统使用**三维评分**对记忆进行排序：

#### 1. 相关性 (Relevance)
- 来源: 向量搜索的余弦相似度分数
- 计算: 已由向量数据库提供 (`_score`)
- 归一化: 缩放到[0, 1]区间

#### 2. 重要性 (Importance)
- 来源: 创建记忆时LLM评估的分数 (0-9)
- 计算: 记忆的`importance`字段
- 归一化: 缩放到[0, 1]区间

#### 3. 时效性 (Recency)
- 来源: 记忆的最后访问时间
- 计算公式: `0.99 ^ (小时数)`
  ```typescript
  const hoursSinceAccess = (now - memory.lastAccess) / 1000 / 60 / 60;
  const recencyScore = 0.99 ** Math.floor(hoursSinceAccess);
  ```
- 特点: 指数衰减
  - 1小时前: score ≈ 0.99
  - 1天前: score ≈ 0.78
  - 1周前: score ≈ 0.49
  - 1个月前: score ≈ 0.04

#### 综合评分公式

```typescript
overallScore = normalize(相关性) + normalize(重要性) + normalize(时效性)
```

**设计思路**:
- 三个维度等权重相加
- 每个维度先归一化到[0, 1]，确保公平性
- 最终分数范围: [0, 3]

**排序与返回**:
```typescript
memoryScores.sort((a, b) => b.overallScore - a.overallScore);
const accessed = memoryScores.slice(0, n);  // 返回top-n
```

### 为什么Overfetch?

```typescript
const MEMORY_OVERFETCH = 10;
limit: n * MEMORY_OVERFETCH  // 获取10倍候选数量
```

**原因**:
1. **向量搜索仅考虑相关性**，不考虑时效性和重要性
2. **需要更多候选**才能在综合排序后找到最优记忆
3. 例如: 需要3条记忆，先取30条候选，综合排序后选top-3

## 反思机制

### 触发条件

```typescript
// 文件: convex/agent/memory.ts:323-395

const sumOfImportanceScore = memories
  .filter((m) => m._creationTime > lastReflectionTs)
  .reduce((acc, curr) => acc + curr.importance, 0);

const shouldReflect = sumOfImportanceScore > 500;
```

**条件**: 自上次反思以来，累计重要性分数超过500

**举例**:
- 如果平均每条记忆重要性为5，需要累积100条记忆
- 如果有10条重要性为9的记忆，加上其他，可能50条就触发

### 反思生成流程

```typescript
// 第348-358行: 构建提示词
const prompt = [

memories.forEach((m, idx) => {
  prompt.push(`陈述${idx}：${m.description}`);
});
prompt.push('你能从以上陈述中推断出哪3条高层次的见解？');
```

**LLM返回格式**:
```json
[
  {
    "insight": "团队合作比单打独斗更重要",
    "statementIds": [1, 5, 12]
  },
  {
    "insight": "保护地球需要勇气和智慧",
    "statementIds": [3, 7, 18]
  },
  {
    "insight": "和人类建立信任是关键",
    "statementIds": [2, 9, 15]
  }

```

**存储反思** (第371-388行):
- 为每条见解计算重要性
- 生成向量表示
- 存储到`memories`表
- `type: 'reflection'`
- 包含`relatedMemoryIds`字段

### 反思记忆的作用

1. **信息压缩**: 将多条具体记忆压缩为高层见解
2. **长期记忆**: 随着时间推移，具体记忆衰减，但见解保留
3. **知识升华**: 从经历中学习和成长
4. **检索效率**: 搜索时可能直接命中高层见解

## 关键算法

### 1. 重要性评估算法

```typescript
// 文件: convex/agent/memory.ts:244-267

async function calculateImportance(description: string) {
  const { content } = await chatCompletion({
    messages: [{
      role: 'user',
      content: `On the scale of 0 to 9, where 0 is purely mundane
      (e.g., brushing teeth, making bed) and 9 is extremely poignant
      (e.g., a break up, college acceptance), rate the likely poignancy
      of the following piece of memory.
      Memory: ${description}
      Answer on a scale of 0 to 9. Respond with number only, e.g. "5"`,
    }],
    temperature: 0.0,  // 确保评分一致性
    max_tokens: 1,     // 只需要一个数字
  });

  let importance = parseFloat(content);
  if (isNaN(importance)) {
    importance = +(content.match(/\d+/)?.[0] ?? NaN);
  }
  if (isNaN(importance)) {
    importance = 5;  // 默认中等重要性
  }
  return importance;
}
```

**设计特点**:
- 使用LLM评估，理解语义重要性
- `temperature: 0.0` 确保评分稳定
- 容错处理：正则提取数字、默认值

### 2. 归一化算法

```typescript
// 文件: convex/agent/memory.ts:174-183

function makeRange(values: number[]) {
  const min = Math.min(...values);
  const max = Math.max(...values);
  return [min, max];
}

function normalize(value: number, range: [number, number]) {
  const [min, max] = range;
  return (value - min) / (max - min);
}
```

**作用**: 将不同量纲的分数统一到[0, 1]区间
**示例**:
- 相关性: [0.3, 0.9] → normalize(0.7) = 0.67
- 重要性: [2, 8] → normalize(5) = 0.5
- 时效性: [0.01, 0.99] → normalize(0.5) = 0.5

### 3. 时效性衰减算法

```typescript
// 第203-206行
const recencyScore = relatedMemories.map((memory) => {
  const hoursSinceAccess = (ts - memory.lastAccess) / 1000 / 60 / 60;
  return 0.99 ** Math.floor(hoursSinceAccess);
});
```

**数学模型**: 指数衰减 `f(t) = 0.99^t`
**曲线图**:
```
1.0 |●
0.8 |  ●
0.6 |    ●●
0.4 |       ●●●
0.2 |           ●●●●●
0.0 |_________________●●●●●●●●
    0   7  14  21  28  ...天
```

**特点**:
- 符合人类记忆遗忘曲线
- 最近的记忆权重最高
- 逐渐衰减但不会归零

## 性能优化

### 1. 向量索引优化

```typescript
// 文件: convex/agent/schema.ts:40-44

.vectorIndex('embedding', {
  vectorField: 'embedding',
  filterFields: ['playerId'],  // 关键优化
```

**优化点**:
- 向量搜索限制在单个角色范围内
- 避免全局搜索，大幅提升速度
- 使用HNSW算法（近似最近邻）

### 2. 访问时间节流

```typescript
// 第12行 + 第220-222行

const MEMORY_ACCESS_THROTTLE = 300_000; // 5分钟

if (memory.lastAccess < ts - MEMORY_ACCESS_THROTTLE) {
  await ctx.db.patch(memory._id, { lastAccess: ts });
}
```

**优化点**:
- 避免频繁更新数据库
- 5分钟内的访问不更新时间戳
- 减少写操作，提升并发性能

### 3. Embedding缓存

```typescript
// 文件: convex/agent/embeddingsCache.ts

export async function fetch(ctx: ActionCtx, text: string) {
  const textHash = sha256(text);
  const cached = await ctx.runQuery(selfInternal.get, { textHash });

  if (cached) {
    return cached.embedding;
  }

  const { embedding } = await fetchEmbedding(text);
  await ctx.runMutation(selfInternal.insert, { textHash, embedding });
  return embedding;
}
```

**优化点**:
- 相同文本不重复调用Embedding API
- 使用SHA256哈希作为键
- 节省API费用和延迟

### 4. 批量操作

```typescript
// 第371-382行: 异步批处理

const memoriesToSave = await asyncMap(insights, async (item) => {
  const importance = await calculateImportance(item.insight);
  const { embedding } = await fetchEmbedding(item.insight);
  return { description, embedding, importance, ... };
});
```

**优化点**:
- 使用`asyncMap`并行处理多条见解
- 避免串行等待，提升反思速度

## 设计亮点

### 1. 模拟人类记忆机制

| 人类记忆 | AI Town实现 |
|---------|------------|
| 短期记忆 | 当前对话的消息列表 |
| 长期记忆 | memories表中的记忆 |
| 遗忘曲线 | 时效性指数衰减 (0.99^t) |
| 重要记忆优先 | importance权重 |
| 联想记忆 | 向量语义搜索 |
| 反思与整合 | reflection机制 |

### 2. 多维评分机制

不同于简单的向量搜索，系统综合考虑：
- **相关性**: 内容语义匹配
- **重要性**: 事件的情感强度
- **时效性**: 记忆的新鲜程度

这更符合人类回忆的真实情况：
- 我们会想起**相关**的事
- 会优先想起**重要**的事
- 会更容易想起**最近**的事

### 3. 自适应反思机制

- **触发条件动态**: 基于重要性总和，而非固定条数
- **高层次抽象**: 从具体经历提炼通用见解
- **知识累积**: 反思记忆本身也可以被检索使用

### 4. 向量化语义搜索

- **超越关键词**: 理解语义相似性
- **隐式关联**: 可以找到间接相关的记忆
- **跨语言**: 向量空间支持多语言

示例:
```
查询: "如何变得更强"
命中记忆: "与赛罗讨论了训练方法" (虽然没有"更强"二字)
```

### 5. 第一人称记忆

```typescript
// 第45行
content: `你是${player.name}，你刚刚和${otherPlayer.name}结束了一次对话。
请用中文从${player.name}的视角总结这次对话，使用第一人称代词如"我"...`
```

**优势**:
- 每个角色有独立视角
- 同一对话，双方记忆不同
- 更真实的角色塑造

示例:
- 迪迦的记忆: "我和赛罗讨论了守护地球，他的热血让我想起了自己的使命"
- 赛罗的记忆: "我和迪迦交流了战斗经验，他的温柔让我感到钦佩"

### 6. 记忆互联

