# 疯狂动物城 地图设计蓝图（v1.0）

> 项目代号 AI-PetTown。本文档是 80×60 主地图的关卡设计规格与制作记录：
> v0.1 为设计稿（Canvas 蓝图），v1.0 补充了实现落地后的坐标修正与美术管线说明（2026-07-12）。
> 对应的地图数据文件：`data/petTownGraybox.ts`，验证脚本：`scripts/validate_pet_town_map.mjs`。

## 基本参数

| 参数 | 值 |
| --- | --- |
| 地图尺寸 | 80 × 60 瓦片 |
| 瓦片尺寸 | 32 px（最终像素尺寸 2560 × 1920） |
| 坐标系 | 原点在左上角，x 向右，y 向下 |
| 地图结构 | 单张连续大地图 + 开放式剖面室内（无场景切换） |
| 瓦片集 | `public/assets/pet-town-graybox.png`（程序生成，含透明碰撞瓦片） |

### 建筑内部布局

**宠物共享之家**：北侧一排睡眠角落（红/蓝小床、沙发、书柜、花瓶圆桌）；东南餐桌区；西侧落地灯与地毯谈心角；内部主过道至少 2 格宽。

**成长学堂**：两间开放教室（各 4 组双人课桌 + 黑板），中央为讲台（老师课桌）；南门 3 格宽直通操场；操场设滑梯、秋千、沙池。

**图书兴趣馆**：西北高书架阅读区（阅读圆桌 + 扶手椅）；东北画架与手工桌；西南钢琴音乐角；东南作品展示板。

**生活街**：收银柜台、货架与苹果货箱构成点心文具店；街侧设邮筒、路灯与巴士站牌；礼物互动发生在柜台，不做商业商城。

## 3. 碰撞与寻路规则

### 碰撞层规则

| 对象 | 碰撞规则 | 例外 |
| --- | --- | --- |
| 水域与深水岸线 | 全部阻挡 | 木桥与钓鱼栈道可行走 |
| 建筑外墙 | 全部阻挡 | 门口至少 2 格宽 |
| 大型家具 | 床、柜台、书架、课桌阻挡 | 椅子旁保留可交互站位 |
| 树木与果树 | 树干 1–2 格阻挡，树冠只做视觉覆盖 | 避免整块树冠成为大碰撞体 |
| 围栏 | 阻挡 | 每个围合区域至少两个出口 |
| 道路、室内地板、草地 | 可行走 | 活动点周围至少保留 2×2 空间 |
| 地毯、野花、栈道等平面装饰 | 不阻挡 | 渲染在角色脚下 |

### 道路与密度标准

- 主路至少 3 格宽，中央环路 4 格宽；室内主要过道至少 2 格宽。
- 建筑门口至少 2 格宽，门外留 3×3 缓冲区。
- 所有主要区域至少存在两条离开路径。
- 广场和操场须支持五只宠物同时聚集不堵塞。

### 验收标准（已通过）

| 检查项 | 通过标准 | 状态 |

## 4. 美术资产管线（实现记录）

### 制作流程

1. 用图像模型生成与"心愿树广场质量样板"同风格的**像素素材设定板**（4×4 网格、统一浅灰背景）。
2. 用 Python 脚本自动切图：边界洪水填充去背景 → 保留最大连通块去杂点 → 自动裁边，输出透明 PNG。
3. 在 `data/petTownGraybox.ts` 中以 `staticsprites` 摆放道具（坐标/尺寸以瓦片为单位）。
4. 原灰盒碰撞块全部替换为透明的 `COLLISION_ONLY` 瓦片，视觉完全由道具贴图承担。
5. 运行 BFS 连通性验证后部署。

### 资产清单

| 类别 | 位置 | 生成脚本 |
| --- | --- | --- |
| 灰盒瓦片集（地面/墙体/水面） | `public/assets/pet-town-graybox.png` | `scripts/generate_pet_town_graybox_tileset.py` |
| 广场地面（圆形石板） | `public/assets/pet-town/plaza/plaza-ground.png` | `scripts/generate_pet_town_plaza_ground.py` |
| 广场素材（心愿树、舞台、公告板等） | `public/assets/pet-town/plaza/` | `scripts/build_wish_tree_plaza_assets.py` |
| 48 个场景道具（家具/植被/街道等） | `public/assets/pet-town/props/` | `scripts/build_pet_town_prop_assets.py` |
| 5 套宠物行走图与头像 | `public/assets/sprites/pets/` | `scripts/build_pet_character_sprites.py` |
| 素材设定板存档 | `docs/ai-pet-town-props-*.png` 等 | — |
| 整图渲染预览 | `/tmp/pet-town-full-preview.png` | `scripts/render_pet_town_preview.py` |

### 深度排序（Y-Sort）机制

- `convex/aiTown/worldMap.ts` 的 `StaticSprite` 支持可选 `sortY`（地面接触点的像素 y 坐标）。
- 带 `sortY` 的静态道具由 `src/components/PixiMapDecorations.tsx` 渲染，zIndex = 1000 + sortY。
- 角色 zIndex 按脚底 y 坐标动态计算（`src/components/Player.tsx`），从而能自然地从道具前后走过。
- 不带 `sortY` 的贴图（地毯、栈道、野花、广场地面）由 `PixiStaticMap` 渲染在角色下方。

## 5. 出生与回归规则

| 对象 | 坐标/规则 | 设计原因 |
| --- | --- | --- |
| 孩子 | (39, 35) | 位于广场南侧，进入后能立即看到心愿树与多条道路 |
| 专属宠物 | (41, 35) | 与孩子距离近，但不重叠出生 |
| 其他宠物 | 由当前日程决定 | 避免所有角色每天固定站在同一位置 |
| 长时间离线回归 | 保持原位置或移动到合理日程点 | 不要无理由把所有宠物传送回家 |
