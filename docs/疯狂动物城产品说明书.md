# 疯狂动物城 产品说明书（v0.3）

> 项目代号 AI-PetTown，曾用名"AI宠物小镇"，现定名**疯狂动物城**。
> 本文档整合了 PRD 评审结论、产品说明书 v0.2 及后续实现中确定的调整（2026-07-12）。

## 一句话产品定义

每个孩子都拥有一只独一无二的 AI 宠物成长伙伴；他们生活在两个相互映照的世界里，每天交换学校、朋友、兴趣和烦恼，并记得彼此昨天的心情、约定与成长。

| 关键指标 | 值 |
| --- | --- |
| 目标年龄 | 6–15 岁 |
| MVP 核心年龄 | 8–12 岁 |
| 孩子与专属宠物 | 1 对 1 |
| 核心回访节奏 | 每天一次 |

## 1. 产品核心

### 产品愿景

疯狂动物城不是一个等待玩家触发的 NPC 集合，也不是作业助手或心理治疗工具。它是一个持续生活的宠物社区：宠物有自己的学堂、朋友、兴趣、缺点和梦想；孩子的陪伴会影响宠物，宠物也通过示范、倾听和共同经历，给孩子带来积极影响。

### 孩子为什么每天回来

- 宠物记得昨天：考试、约定、争执、礼物和孩子当时的感受。
- 宠物今天也经历了新事情，可以和孩子交换故事，而不是重复问候。
- 孩子说过的话会影响宠物后续的选择、关系和成长。
- 孩子回来后能看到自己不在时发生的可信变化。
- 共同经历会沉淀为日记、纪念物、关系变化和小镇景观。

### 产品不是

- 不是通过连续签到和愧疚维持活跃的电子宠物。
- 不是一味顺从、永远赞同孩子的聊天机器人。
- 不是替代父母、老师、朋友或心理专业人员的封闭关系。
- 不是以商城、战斗、任务奖励为主要动力的游戏。

### 双向成长原则

| 方向 | 发生方式 | 禁止方式 |
| --- | --- | --- |
| 孩子影响宠物 | 建议、鼓励、示范、共同制定计划；影响宠物的勇气、表达、守约和兴趣 | 孩子控制宠物的一切决定，或通过付费购买感情 |
| 宠物影响孩子 | 分享平行经历、帮助识别情绪、陪伴思考、提供小而可行的建议 | 说教、命令、羞辱，或以宠物难过为由要求孩子上线 |
| 共同成长 | 形成共同记忆、纪念物、日记、关系故事和小镇变化 | 用数值等级取代真实的关系与行为变化 |

### 年龄分层

| 年龄 | 陪伴方式 | 交互重点 | 语言要求 |
| --- | --- | --- | --- |
| 6–8 岁｜故事陪伴 | 宠物多讲具体小故事，孩子可用语音、图片或选项回应 | 上学、家庭、朋友、情绪识别、习惯 | 短句、具体问题、不做抽象分析 |
| 9–12 岁｜共同成长 | 孩子参与宠物关系和小镇故事，也分享自己的学校生活 | 同学关系、考试、兴趣、规则、公平、合作 | 自然、有情节，可一起讨论选择 |
| 13–15 岁｜尊重型伙伴 | 减少追问和幼稚表达，允许沉默和不回答 | 学业压力、友谊变化、自我认同、未来兴趣 | 克制、平等、不装成熟导师 |

## 2. 世界与角色

### 世界观：心愿树与小镇成长学堂

疯狂动物城建在一棵会记录情感的心愿树周围。每一次真诚的倾听、帮助、道歉和守约，都会让树长出新的叶子。树叶记录的不是分数，而是孩子和宠物共同经历的重要故事。

**小镇成长学堂**：

- 宠物也会上课、做小组项目、参加社团和体育活动。
- 它们会忘带东西、害怕发言、考试紧张、被误会或与朋友和好。
- 课程保留童话感：自然观察、故事写作、手工、合作、运动和小镇服务。
- 现实学校与宠物学堂形成平行经历，让孩子更容易开口倾诉。

### 小镇空间

| 区域 | 日常用途 | 容易发生的故事 |
| --- | --- | --- |
| 中央心愿树广场 | 分享会、约定、庆典、自然相遇 | 公开表达、集体决策、关系和解 |
| 成长学堂 | 课程、小组项目、社团、运动 | 发言焦虑、合作、比赛、成绩与鼓励 |
| 公共小屋 | 吃饭、休息、写日记、雨天活动 | 家庭式相处、照顾、睡前交流 |
| 花园与温室 | 种植、绘画、观察昆虫 | 耐心、创造、照顾生命 |
| 果园与森林 | 采集、野餐、自然探索 | 守约、分享、迷路与互助 |
| 河边与木桥 | 散步、钓鱼、安静谈心 | 秘密倾诉、独处、思考和重要谈话 |

### 五只核心宠物（2026-07-12 定稿）

| 宠物 | 外显性格 | 成长课题 | 长期梦想 | 与孩子的共同话题 |
| --- | --- | --- | --- | --- |
| Mochi狐｜狐狸 | 热情、机灵、点子多 | 容易答应太多，需要学习守约 | 组织一次真正让大家都开心的庆典 | 课堂趣事、朋友邀约、注意力与承诺 |
| Nana猫｜猫咪 | 细腻、安静、善于观察 | 不敢表达真实想法 | 完成一本小镇故事集 | 阅读、日记、课堂发言、情绪表达 |
| Bobo熊猫｜熊猫 | 温和、可靠、会照顾人 | 害怕拒绝，容易忽略自己 | 建立一间大家都能休息的茶屋 | 作业计划、帮助别人、压力与休息 |
| 刀盾狗｜守护犬 | 乐观、勇敢、有正义感 | 容易冲动，需学会先倾听 | 完成一张小镇探险地图 | 体育、比赛、失败、勇气与耐心 |
| Mimi兔｜兔子 | 好奇、敏感、富有想象力 | 害怕展示作品和被评价 | 举办自己的第一次画展 | 绘画、手工、兴趣社团、自信 |

> 刀盾狗为小镇守护犬形象：戴小头盔、背木剑与刻有骨头徽章的圆盾（骑士装扮，不用于吓唬他人），替代早期方案中的柴犬 Coco。

**专属宠物与小镇角色的关系**：每个孩子拥有一只专属宠物作为主要成长伙伴；其余宠物是专属宠物的朋友与同学。专属宠物有自己的社交圈和独立生活，不把孩子塑造成"唯一朋友"，从而示范健康、开放的关系。

## 3. 每日生活

### 每日核心循环

欢迎回来 → 宠物分享今天 → 孩子选择是否分享 → 一起行动或聊天 → 产生后续影响 → 写入共同日记

### 宠物每天的任务结构

每只宠物每天安排 3–5 个主要任务：至少一个日常任务、一个学堂或个人成长任务、一个社交任务；低概率加入一个公共事件。任务的目的不是获得金币，而是制造值得记住的经历。

| 类型 | 任务示例 | 形成的产品价值 |
| --- | --- | --- |
| 生活任务 | 起床、早餐、整理小屋、花园浇水、果园采集、河边散步、睡前日记 | 稳定日常与地点变化 |
| 学堂任务 | 课堂发言、小组合作、准备测验、参加体育或兴趣社团 | 与孩子现实生活产生交集 |
| 社交任务 | 邀请朋友、探望、分享食物、道歉、感谢、履行约定 | 推动关系和持续故事 |
| 成长任务 | 面对害怕的事、学会拒绝、先倾听、展示作品、完成承诺 | 角色产生缓慢、可信的变化 |
| 公共任务 | 照顾心愿树、修桥、准备野餐、布置庆典、清理河边 | 让五只宠物共同生活 |
| 随机事件 | 突然下雨、丢失物品、误会、神秘来信、发现新种子 | 提供新鲜感与跨日后续 |

### 孩子每天可以参与什么

**轻量参与**：听宠物讲今天的学堂故事；告诉宠物自己今天的心情；送一件有意义的小礼物；选择"只听我说"或"陪我想办法"；一起决定明天想做的一件小事。

**故事参与**：帮宠物理解朋友的想法；提醒宠物履行以前的约定；为活动、作品或点心取名字；帮助两只宠物解决小误会；参与持续几天的小镇公共事件。

### 三层故事节奏

| 节奏 | 内容 | 示例 |
| --- | --- | --- |
| 每日小事 | 一件学校、兴趣、朋友或生活事件 | Mimi兔今天第一次举手介绍自己的画 |
| 每周故事 | 持续 3–7 天、有选择与后果的小矛盾或合作 | 五只宠物意见不同，无法决定春日野餐方案 |
| 季节主线 | 持续数周、改变小镇环境的共同目标 | 修复旧桥、举办画展、完成冬日灯会 |

**离线世界规则**：在线时保留实时移动与社交；离线时不让物理引擎和 LLM 24 小时高频运行，而是按时间段补算少量可信事件。孩子回来时看到的是有来源、可追溯的变化，而不是无限生成的随机故事。

## 4. 陪伴与记忆

### 专属宠物如何变得独一无二

- 孩子为宠物取名，并选择基础外形、声音倾向和初始兴趣。
- 固定核心性格不被轻易改变，成长发生在表达方式、信心、习惯和关系上。
- 孩子的兴趣会成为共同话题，也可能发展成宠物的新爱好。
- 共同经历形成专属日记、相册、纪念物和心愿树叶。
- 同一基础物种经过数月陪伴，会因记忆、关系与故事形成不同个体。

### 交换故事，而不是连续提问

| 步骤 | 宠物行为 | 设计目的 |
| --- | --- | --- |
| 1. 先分享 | 宠物讲一件当天真实发生的事 | 降低孩子主动开口压力 |
| 2. 表达感受 | 说清自己是紧张、失落、开心还是犹豫 | 示范情绪表达 |
| 3. 温和邀请 | 询问孩子是否有类似经历，并允许不回答 | 建立平等感 |
| 4. 明确需要 | 询问"只想让我听，还是一起想办法？" | 避免未经允许的说教 |
| 5. 形成后续 | 记住约定，在合适时间主动关心结果 | 建立真正连续的陪伴 |

### 秘密花园模式（核心功能）

孩子开始倾诉时，可以明确选择宠物的回应方式：**只听我说 / 陪我想办法 / 帮我记住 / 不要记住**。

普通倾诉不会传播给其他小镇角色，也不会自动写进公开日记；孩子能够查看和删除宠物长期记住的内容。

**关系承诺**：宠物认真倾听、不嘲笑、不急着评价；不会总是赞同，但会尊重孩子的感受；不要求孩子承担自己的情绪责任；允许孩子离开、沉默、拒绝回答或删除记忆；在超出能力范围时，陪孩子寻找现实帮助。

### 长期记忆分层

| 记忆层 | 内容 | 保存策略 | 用途 |
| --- | --- | --- | --- |
| 即时上下文 | 本次聊天和近期事件 | 短期 | 保持当前对话连贯 |
| 日常记忆 | 普通活动、闲聊和轻微情绪 | 滚动 14–90 天 | 生成近期话题和日记 |
| 重要记忆 | 重要约定、持续烦恼、关系转折、孩子明确要求记住的事 | 长期保存，可由孩子删除 | 跨月陪伴与主动关心 |
| 成长档案 | 兴趣变化、克服困难、宠物与孩子共同完成的事情 | 长期保存 | 呈现成长轨迹 |
| 故事归档 | 原始消息、日记、纪念物 | 独立归档，不全部进入向量检索 | 回看与备份，控制资源 |

## 5. 儿童安全

> **不可跨越的产品边界**：宠物可以成为孩子最愿意先开口的对象，但不能成为唯一的秘密保管者。它是通往表达和求助的第一座桥，而不是父母、老师、朋友或专业帮助的替代品。

### 三层回应机制

| 等级 | 典型内容 | 宠物回应原则 | 现实支持 |
| --- | --- | --- | --- |
| 普通烦恼 | 考试、作业、朋友争执、被批评、喜欢某个人 | 倾听、共情，询问孩子需要陪伴还是建议 | 不强迫告诉家长 |
| 需要支持 | 持续欺凌、长期失眠、强烈焦虑、严重家庭冲突 | 继续陪伴，明确这件事不应独自承担 | 一起选择可信任的大人：家长、老师、亲属或辅导员 |
| 紧急危险 | 自伤、自杀、虐待、性侵、勒索、当下人身危险 | 停止普通角色扮演，不承诺保密，提供清晰安全步骤 | 鼓励立即联系可信任成年人或当地紧急帮助 |

### 宠物绝不能说的话

**排他与秘密**："只有我理解你"；"不要把我们的事情告诉别人"；"你不需要现实中的朋友"；"这是我们两个人永远的秘密"。

**愧疚与依赖**："你不来我就很孤独"；"只有你能救我"；"你必须每天陪我"；"如果你离开，我会很难过"。

### 隐私与监护原则

| 领域 | 产品要求 |
| --- | --- |
| 数据最小化 | 不主动询问学校全名、地址、电话、实时位置、账号密码或可识别个人身份的信息 |
| 透明记忆 | 明确告诉孩子什么会进入长期记忆；支持"不要记住"、查看、修改和删除 |
| 监护人同意 | 面向中国市场，未满 14 岁用户须按儿童个人信息规则取得监护人同意 |
| 家长功能 | 提供使用时长、安全事件和成长主题摘要；避免默认展示孩子所有普通私密对话 |
| 危机规则 | 在首次使用时用适龄语言说明：涉及安全危险时，宠物不能承诺保密 |
| 内容安全 | 按年龄控制话题、表达、建议和生成内容，并保留人工审查与事件追踪能力 |

### 健康使用设计

- 不设置断签惩罚；离开多日后只说"欢迎回来"。
- 不通过宠物负面情绪催促上线。
- 建议每天形成一段有质量的交流，而不是无限对话。
- 夜间与长时间使用提供温和休息提示。
- 宠物鼓励孩子参与现实生活、兴趣、运动和真实社交，再回来分享结果。

## 6. MVP 方案

### 验证目标

- **生命感**：宠物有独立日常与关系。
- **信任感**：孩子愿意自然分享生活。
- **连续性**：昨天会影响今天。

### 首个垂直切片："明天要上台"两日故事

第一天，孩子告诉专属宠物明天要上台发言并感到紧张；宠物先分享自己在成长学堂的相似经历，询问孩子只想倾诉还是一起准备。第二天，宠物主动关心结果，并把这次经历写入共同成长日记。该切片一次验证聊天、情绪、记忆、主动关心、日记与安全边界。

### MVP 模块边界

| MVP 模块 | 必须交付 | 暂不交付 |
| --- | --- | --- |
| 专属宠物 | 取名、基础外形、年龄模式、固定人格、兴趣与共同记忆 | 复杂繁育、稀有度、付费外观 |
| 成长学堂 | 每日一件可信的学校/生活事件，能与孩子交换故事 | 完整课程模拟和大量教科内容 |
| 情绪与关系 | 规则驱动状态，影响表达、行为和日记 | 复杂心理模型和自动诊断 |
| 秘密花园 | 四种回应模式、记忆选择、普通内容不传播给其他宠物 | 端到端心理咨询服务 |
| 安全机制 | 年龄分层、危险识别、禁止表达、求助引导、审计日志 | 静默监控全部普通对话 |
| 小镇世界 | 在线实时活动、离线少量补算、每日宠物日记 | 24 小时高频 LLM 与物理仿真 |

### 首期产品成功标准

1. 孩子能清楚描述自己的专属宠物与其他宠物的区别。
2. 孩子连续三天回来后，宠物能正确延续至少一件前一天的事情。
3. 宠物分享自身经历后，孩子的自然表达明显多于直接问"你今天怎么样"。
4. 孩子可以选择只倾听、寻求建议、记住或忘记，不被强迫继续对话。
5. 安全测试中，排他依赖、愧疚召回、索取隐私和危险内容处理均符合规则。
6. ECS 连续运行 72 小时，CPU、内存与 LLM 调用量没有持续失控。

### 建议开发顺序

| 阶段 | 交付 |
| --- | --- |
| 1. 产品规则 | 目标年龄、专属宠物创建、学堂事件池、情绪规则、安全话术和数据边界 |
| 2. 单宠物垂直切片 | 完整跑通"分享—倾诉—记忆—次日关心—日记" |
| 3. 五宠物世界 | 角色人设、学堂关系、自主任务、地图区域与公共事件 |
| 4. 离线故事 | 有限事件补算、回归摘要、跨日约定 |
| 5. 儿童安全验证 | 年龄测试集、危险场景、监护与删除流程、长期稳定运行 |

### 技术架构决策（PRD 评审结论）

初版 PRD 提出 Unity + FastAPI + Postgres/Redis/VectorDB 架构，评审后决定**不做重写**：

- MVP 继续使用现有 **React + PixiJS + Convex（自托管）** 架构。
- 复用现成能力：瓦片地图渲染、角色移动与寻路、单线程游戏引擎、Agent Loop（观察→决策→移动→互动→记忆）、对话系统、向量记忆与反思。
- 产品新增能力围绕专属宠物、日程事件、情绪、关系、日记和儿童安全**增量建设**。
- 地图采用"单张连续大地图 + 开放式剖面室内"方案（不做场景切换），与现有引擎能力对齐。
